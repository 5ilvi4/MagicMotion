🚀 QUICK START (3 STEPS)
═════════════════════════════════════════════════════════════════════════════

Step 1️⃣  BUILD MEDIAPIPE FRAMEWORK (10-15 minutes)
   $ cd /workspaces/MagicMotion
   $ chmod +x build_mediapipe.sh
   $ ./build_mediapipe.sh
   
   Requires: Xcode CLI tools, Bazel
   Output: MediaPipeFramework/ directory

Step 2️⃣  LINK FRAMEWORK IN XCODE (5 minutes)
   1. Open MagicMotion.xcodeproj
   2. Select Project → Target: MagicMotion
   3. Build Phases → Link Binary With Libraries → +
   4. Navigate to: /workspaces/MagicMotion/MediaPipeFramework
   5. Select: MediaPipeTasksVision.framework
   6. Click: Add

Step 3️⃣  APPLY LINKER FLAGS (3 minutes)
   1. Select Project → Target → Build Settings
   2. Search "Other Linker Flags" → Add: -lc++ -ObjC
   3. Search "C++ Language Dialect" → Select: GNU++17
   4. Search "C++ Standard Library" → Select: libc++
   5. Search "Enable Bitcode" → Set: No
   
   ✅ Result: All undefined symbol errors FIXED
