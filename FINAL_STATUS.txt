================================================================================
                            ARCHITECTURE FLOW
================================================================================

Camera Frame (Raw)
    ↓
FrameSource.onNewFrame callback (30fps)
    ↓
MotionEngine.processFrame(buffer, orientation)
    ↓
MediaPipe extracts 33 landmarks → PoseSnapshot emitted
    ↓
MotionInterpreter receives PoseSnapshot
    ├─ Confidence gating (only if reliable: confidence > 0.5)
    ├─ Ring buffer smoothing (15 frames, ~0.5s at 30fps)
    ├─ Event classifiers (handsUp, jump, squat, leanLeft/Right, handsDown, freeze)
    ├─ 3-frame confirmation gate (event fires only if 3 consecutive frames)
    └─ 500ms cooldown (prevent spam between events)
    ↓
MotionEvent fires (.jump, .squat, .leanLeft, etc.)
    ↓
GameSession.handle(event:)
    ├─ Maps event to player action
    ├─ Updates player state
    └─ Runs game loop (collision detection, scoring)
    ↓
GameView renders updated FSM state
    ├─ idle → START button
    ├─ calibrating → spinner
    ├─ countdown(3) → big countdown
    ├─ active → game loop + HUD
    ├─ paused → resume/reset buttons
    ├─ roundOver → game over screen
    └─ completed → congratulations
    ↓
ContentView layouts:
    ├─ iPad + TV connected: iPad shows operator panel, TV shows GameView
    └─ iPad only: iPad shows both camera + embedded GameView
    ↓
Rendered UI on iPad / TV

TOTAL END-TO-END LATENCY: ~100ms (typically < 50ms)

================================================================================
                          KEY FEATURES IMPLEMENTED
================================================================================

✅ Protocol-Based Abstraction
   - FrameSource protocol allows swapping CameraManager ↔ SyntheticFrameSource
   - Easy to test without camera/MediaPipe

✅ Confidence Gating
   - Only classify poses if snapshot.isReliable (confidence > 0.5)
   - Noise reduction: prevents false event firings

✅ Ring Buffer Smoothing
   - 15 frames (~0.5s at 30fps) stored in rolling buffer
   - Allows statistical analysis: handsUp confirmed if 3 consecutive frames

✅ 3-Frame Confirmation Gate
   - Event must appear 3 consecutive frames before firing
   - Eliminates false positives from single noisy frames

✅ 500ms Cooldown
   - Prevents rapid-fire events
   - Example: handsUp for 2s → fires at 0ms and 500ms (not every 30ms)

✅ FSM-Based Game State
   - 7 explicit states: idle, calibrating, countdown, active, paused, roundOver, completed
   - Game loop only runs when state == .active
   - Clear transitions prevent undefined behavior

✅ MediaPipe Isolation
   - ONLY MotionEngine imports MediaPipeTasksVision
   - All other layers use PoseSnapshot (no MediaPipe knowledge)
   - Easy to replace MediaPipe with another pose detector

✅ Multi-Screen Support
   - ExternalDisplayManager detects TV connection
   - Mounts GameView on TV (kid-facing)
   - Shows operator panel on iPad (parent controls)

✅ Testing Framework
   - FakeMotionSource replays scripted poses (6 gesture fixtures)
   - DebugOverlayView shows landmarks, event labels, FPS (#if DEBUG)
   - No camera needed for testing
